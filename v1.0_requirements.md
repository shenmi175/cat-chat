# “没头脑的马屁精” 桌面互动小宠物 v1.0 需求与开发文档

## 1. 产品概述
- **产品名称**：没头脑的马屁精
- **产品定位**：一款运行在电脑桌面上的虚拟宠物（小猫形态）。它的性格有些笨笨的，但非常喜欢拍用户马屁，能在用户工作或娱乐时提供情绪价值。
- **核心交互**：在桌面上随时可见、可自由拖拽；能够接入 DeepSeek API 进行对话；**核心亮点在于“桌面状态感知”**，它能根据当前系统状态（如电量、时间、当前运行的活跃窗口等）主动且有代入感地做出反应或吐槽（拍马屁）。

---

## 2. 技术栈选择：Electron + React (Vite)

选择该方案由于其极高的成熟度和容错率，前端丰富强大的生态能让你轻易写出好看的气泡UI并控制动画，Electron能在后台静默调用丰富的Node能力读取系统状态。

### 核心依赖：
- **Electron**: 提供桌面端窗口（无边框、透明背景、置顶）以及 Node.js 运行时环境。
- **React + Vite**: 提供界面的极速渲染、组件化开发体验。
- **axios / fetch**: 用于向 DeepSeek 发送请求。
- **active-win**: 用于获取当前系统最上层的活跃窗口（从而知道用户在用什么软件）。
- **systeminformation**: 用于获取系统的 CPU、内存、电池状态等底层信息。

---

## 3. 功能规划 (v1.0 Demo)

### 3.1 核心需求
1. **交互形态与视效**：
   - 展现为一只无边框、背景完全透明的“小猫”窗口。
   - 窗口始终置顶。
   - 可以通过鼠标按住小猫自由拖动。
2. **状态感知能力 (主动触发)**：
   - **时间感知**：根据当前时间（如深夜2点）自动弹气泡拍马屁（“哇，主人这么晚还在工作，简直是人类高质量打工人！”）。
   - **电量感知**：电脑低电量时触发（“主人，本喵快饿晕了，你的电脑好像也快没电了喵~”）。
   - **运行软件感知**：通过 `active-win` 发现用户当前打开了特定软件（如 VSCode 或网页）时，小猫主动弹气泡（“主人又在写代码了？虽然我看不懂，但我觉得这行代码价值百万！”）。
3. **基于 DeepSeek API 的对话能力 (被动触发)**：
   - 点击小猫后，可以弹出一个气泡输入框，键入文字进行对话。
   - 将用户的话与系统收集到的“当前软件状态”一并发送给 DeepSeek，让 DeepSeek 结合情境进行恢复。
   - **Prompt 设计**：严格将其性格限制在“没头脑的马屁精”，字数限制在 50 字以内。

## 4. 安装与运行说明 (Demo 环境)

开发前需要在本机安装 [Node.js](https://nodejs.org/) (建议 LTS 版本)。

### 4.1. 依赖安装与启动
在终端中进入项目目录，运行以下命令安装依赖并启动项目：
```bash
# 1. 安装项目所有依赖
npm install

# 2. 一键启动开发环境 (同时启动 Vite 前端 + Electron 后端)
npm run dev
```

### 4.2. 一键卸载所有环境依赖
如果你希望完全清理该项目产生的依赖文件（如 `node_modules`）恢复干净状态，只需在项目根目录下运行：
```bash
# Linux / macOS
rm -rf node_modules package-lock.json

# Windows (PowerShell)
Remove-Item -Recurse -Force node_modules, package-lock.json
```
清理后，项目除了源码外不会在系统中残留任何环境依赖，随时可以重新 `npm install`。

## 5. 接下来要做的实现步骤

1. 使用 Vite 初始化 React 项目 `npm create vite@latest . -- --template react`。
2. 安装 Electron 及相关工具 `npm install electron vite-plugin-electron -D`。
3. 编写 `electron/main.js` 配置无边框、透明、可鼠标穿透的窗口。
4. 编写预加载脚本 `electron/preload.js` 暴露系统状态信息（`active-win`、系统时间与电量）给 React 前端。
5. 在 React 中编写小猫 UI，加入可拖拽逻辑和聊天气泡组件。
6. 最后编写 DeepSeek API 的调用逻辑（含定时轮询系统状态与主动触发发话的机制）。
